# Support

Use GitHub Issues for reproducible bugs and feature/package proposals. Include the
package and version/commit, Unity version, render pipeline, platform, minimal
reproduction, logs, and expected versus actual behavior.

Security reports must follow [SECURITY.md](SECURITY.md), not a public issue.

Use the [Task Hall](docs/contributing/task-hall.md) for bounded public contribution
work. Use the [Device Lab](docs/device-lab/README.md) and Device test form for
headset evidence; general support reports and Editor screenshots are not device
receipts.

Incubating packages are best-effort and may change. Candidate/stable support terms
will be stated in their release notes.
