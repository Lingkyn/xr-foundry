# Public work map

This is the maintained human entry to planned and proposed XR Foundry work. The
machine projection is [`public-work-map.json`](public-work-map.json); detailed
checkpoint authority remains in the registered task contracts under
[`tasks/`](tasks/).

## Two entry routes

| Origin | Starts at | What happens next |
| --- | --- | --- |
| Maintainer Plan | [`ROADMAP.md`](../../ROADMAP.md) | A maintainer shapes an outcome into an umbrella, checkpoints, acceptance, and routing requirements. |
| Community Proposal | [Reusable-system proposal](https://github.com/Lingkyn/xr-foundry/issues/new?template=package-proposal.yml) | Public source and recurrence evidence is reviewed before any package identity or Ready task exists. |

Both routes converge at **maintainer admission**. Neither a roadmap row, proposal,
comment, Agent plan, or Project card grants Ready status, repository permission,
or merge authority.

## Which surface answers which question?

| Surface | Question it answers |
| --- | --- |
| Roadmap | What outcomes should happen, and in what broad order? |
| Milestone | Which outcomes form one reviewable result batch? |
| Project | What is currently scheduled, Ready, active, waiting, or integrating? |
| Umbrella issue | What is the shared outcome and dependency graph? |
| Checkpoint issue | What bounded unit can someone discuss or claim? |
| Task contract | What are the exact paths, gates, routing, acceptance, evidence, and next action? |
| Task registry | Which task contracts are current public authority? |
| Pull request | What exact diff is proposed for integration? |
| Evidence receipt | What exact claim was reproduced for which commit, environment, or device? |

## Fresh-context continuation

Start from the [live Project](https://github.com/users/Lingkyn/projects/2) or the
table above, open one umbrella, then read its registered task contract. Work only
on one named checkpoint. Before material changes, obtain the maintainer-confirmed
claim and publish the required execution anchor. Before stopping, either complete
the checkpoint or publish its continuation receipt. Private chat history is never
required to resume public work.
