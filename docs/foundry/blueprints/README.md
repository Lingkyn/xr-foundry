# Admitted package blueprints

This directory records package identities only after the positive-source gate,
public implementation Issue, and independent advisory source review are complete.
Each file is a deterministic input to the Foundry scaffolder; it grants no catalog,
maturity, release, compatibility, or device status.

An Agent can resume one blueprint independently. It must preview the scaffold,
write only to the declared empty target, replace the deliberate failing scaffold
test with focused implementation and tests, and publish a reachable checkpoint
before beginning a sibling package.
